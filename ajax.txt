◎파라미터 -> GET파라미터, POST파라미터, PUT파라미터, UPDATE파라미터, ...

form -> method 속성 : GET, POST

파라미터는 서버로 보내는 데이터

-GET -> 보내는 데이터가 URL에 그대로 노출이 됨 / 이 방식으로 아이디, 비밀번호 같은 민감한 개인정보를 보내면 그 민감한 개인정보가 URL에 그대로 노출이 됨

주로 노출이 되도 상관없는 민감하지 않은 데이터를 보낼 때 사용

-POST -> 보내는 데이터가 URL에 노출되지 않음 / 민감한 개인정보를 보내도 URL에 그대로 노출되지 않으니까 민감한 개인정보를 보낼 때 사용

GET, POST의 또 다른 차이점 -> 보낼 수 있는 데이터의 크기가 다름

GET은 보낼 수 있는 데이터의 크기가 한정되어 있음

POST는 보낼 수 있는 데이터의 크기가 한정되어 있지 않음

그래서 GET은 간단한 데이터를 서버로 보낼 때 사용

POST는 파일처럼 크기가 크거나 복잡한 데이터를 서버로 보낼 때 사용

-PUT

-DELETE
